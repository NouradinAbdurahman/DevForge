## dist-tags at publish time

```json
{ "next": "3.0.1-rc1", "latest": "3.0.1-rc1" }
```

**Known issue, flagged immediately**: npm always assigns `latest` to a
package's very first published version, regardless of `--tag` - there
is no prior stable version for `latest` to keep pointing at. This means
a plain `npm install -g devforgekit` (no `@next`) would install the RC
right now, which is not the intent (the RC should only be reachable via
`devforgekit@next` until a real `3.0.1` stable ships). See the
correction note below once resolved.
